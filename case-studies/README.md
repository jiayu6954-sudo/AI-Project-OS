## 跨项目高频踩坑 TOP 10

> 这些坑在多个项目中重复出现，AI 生成代码时特别容易犯。

| 排名 | 问题 | 出现项目 | 对应清单项 |
|------|------|---------|----------|
| 🔴 1 | 有状态对象（KieSession/连接）作为单例注入 | 医保 | 清单 1.1 |
| 🔴 2 | 硬编码密钥或可预测随机数生成 API Key | CDN | 清单 6.1/6.2 |
| 🔴 3 | Service 接口有声明无实现，启动即崩溃 | 医保 | 清单 5.6 |
| 🔴 4 | 业务决策结果被 @Cacheable 缓存 | 医保 | 清单 2.1 |
| 🟡 5 | Mock 对象无 stub 配置，测试必然失败 | 医保、Seed AI | 清单 8.1 |
| 🟡 6 | `import` 位置错误或缺失（Go/Java 各有变种） | CDN、医保 | 清单 5.3 |
| 🟡 7 | 同包内重复定义同名函数，编译冲突 | CDN | 清单 5.4 |
| 🟡 8 | Java `assert` 代替 `Assertions`，断言永不执行 | 医保 | 清单 8.2 |
| 🟡 9 | 枚举字段缺少 @EnumValue，按字符串存整型列 | 医保 | 清单 5.5 |
| 🟡 10 | 内部 LLM 调用绕过 Provider 接口直接 new SDK | Seed AI | 清单 3.1 |

## 如何归档新案例

1. 项目完成后，将 `/memory/项目操作记忆.md` 复制到本目录
2. 命名：`[项目名]-[YYYY-MM].md`
3. 在上方"案例列表"表格中追加一行
4. 检查"高频踩坑"表：新项目有没有新坑需要追加？
