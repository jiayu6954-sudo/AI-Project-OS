## 一、项目定义书

**系统定位**：构建工业级 CDN 边缘计算基础设施，包含控制面板（Control Plane）、数据面（Data Plane）和完整的运维工具链。

**项目背景**：需要一套可自托管的 CDN 系统，支持域名管理、边缘节点分发、WAF 防护和实时监控。控制面板负责配置管理，数据面（OpenResty）负责流量处理。

### 硬性约束

| 类别 | 约束内容 | 量化指标 |
|------|---------|---------|
| 性能 | API 响应时间 | P95 ≤ 100ms |
| 吞吐量 | 单节点 | > 10,000 QPS |
| 吞吐量 | 集群 | > 100,000 QPS |
| 可用性 | 系统可用性 | > 99.99% |
| 恢复 | 故障恢复时间 | < 5 分钟 |

### 成功标准

- [ ] 控制面板（Go）可编译，核心 API 全部实现
- [ ] 数据面（OpenResty + Lua）边缘节点可部署

- [ ] Docker Compose 一命令启动完整环境
- [ ] 工具链完整（构建/部署/监控/运维）
- [ ] 运维手册和 API 文档完整

### 技术栈决策

| 组件 | 最终选型 | 核心原因 | 主要备选 |
|------|---------|---------|---------|
| 控制面板 | Go 1.21 + Gin + PostgreSQL + Redis | 高性能、并发性好、部署简单（单二进制） | Java Spring Boot |
| 数据面 | OpenResty 1.21.4.1 + LuaJIT | Nginx 生态 + Lua 扩展，CDN 业界标准方案 | Nginx + C 模块 |
| Lua 共享内存 | `ngx.shared.DICT` | 进程间通信，配置同步，零外部依赖 | Redis Pub/Sub |
| 监控 | Prometheus + Grafana + Loki | 监控标准化，生态完善 | 自研监控 |
| 部署 | Docker + Docker Compose | 环境一致性，简化部署 | 裸机部署 |
| 构建 | Makefile | 跨平台，命令统一 | Shell 脚本 |

### 排除项

- 不做 DNS 解析服务（用户自行配置 CNAME）
- 不做付费计费系统（纯技术基础设施）

## 二、关键设计决策

| # | 决策点 | 最终选择 | 放弃的方案 | 决策理由 |
|---|--------|---------|-----------|---------|
| D-1 | 认证架构 | 双重认证：JWT（用户 API）+ API Key（节点 API） | 统一 JWT | 节点心跳不适合携带 JWT（无 UI 登录），API Key 更适合机器间认证 |
| D-2 | 配置同步机制 | 基于版本号（`config_version` 字段比较） | 全量推送 | 版本号比对避免无效同步，节省带宽，节点可判断是否需要更新 |
| D-3 | Node-Agent 同步策略 | `sync_interval × (1 ± jitter)` 随机抖动 | 固定间隔 | 大规模节点（100+）同时拉取会导致控制面板瞬时过载（惊群效应） |
| D-4 | WAF 规则匹配引擎 | `ngx.re.find`（PCRE JIT 编译） | `string:match`（Lua 原生） | `ngx.re` 支持 PCRE JIT，性能比 Lua 原生 `match` 高 10 倍+ |
| D-5 | API Key 格式 | 前缀 `cdn-` + `crypto/rand` 32字节 | 无前缀时间戳 | 前缀便于日志审计时快速识别密钥类型；crypto/rand 保证密码学安全 |
| D-6 | migrations 策略 | 保留 `migrations.go`（嵌入式SQL），删除 `database.go` | 两者共存 | migrations.go 使用嵌入式 SQL（无外部文件依赖），是唯一权威源；database.go 是冗余实现 |

## 七、经验教训

1. **Go import 位置**：AI 在追加新 import 时常写到文件末尾，必须在约束提示词中强调 import 必须在文件顶部
2. **同包重名函数**：合并多个文件实现时必须先检查函数名冲突，`go build` 会给出明确报错但 AI 不总能预判
3. **密钥必用 crypto/rand**：时间戳、UUID 均不具备密码学安全性；CDN API Key 的安全性直接影响节点认证
4. **单文件行数上限**：handlers.go 超过 1000 行时，模型名不一致等错误积累且难以发现；300 行是合理上限
5. **node-agent 等子系统必须完整实现**：骨架引用不存在的包是比较隐蔽的问题，编译时才暴露
6. **WAF 必须防绕过**：编码绕过（URL/HTML实体编码）是 WAF 最常见攻击手法，规则匹配前必须做标准化处理
7. **容器内 DNS 显式配置**：Nginx 在容器内解析域名时需要显式指定 `resolver 127.0.0.11`（Docker 内置 DNS）
